# EE3220 TBL-2 — RISC-V Mission Clock (PicoRV32 SoC)

## Overview

Bare-metal firmware on a **PicoRV32** SoC drives a **mission clock** (`hh:mm:ss`, 24-hour rollover) on a **UART TX** line. A **memory-mapped timer** delivers **1 Hz** ticks (in terms of the SoC clock configuration); firmware **polls** `TIMER_STATUS` and uses **write-1-to-clear** to acknowledge ticks—**no interrupts**.

Instruction memory is initialized from **`clock.mem`** (compiled firmware). Data memory holds stack and `.data`/`.bss`. **Simulation** uses accelerated clock/timer parameters in `tb_basic`; **FPGA** uses `pynq_z2_tx_demo_top` with a **PLL** to the SoC clock and **115200 8N1** UART.

## MMIO map

Address decoding and `bus_valid` gating are implemented in **`picorv32_soc_ref.sv`** (UART and Timer each occupy a 16-byte window).

| Peripheral | Base | Offset | Access | Description |
|------------|------|--------|--------|-------------|
| **UART** | `0x4000_0000` | `+0x00` | WO | **TXDATA** — write low 8 bits to send; if the UART is busy, the bus stalls until ready. |
| | | `+0x04` | RO | **STATUS** — bit [0] `tx_ready`: `1` = idle, `0` = transmitting. |
| **Timer** | `0x4000_0010` | `+0x00` | RW | **STATUS** — bit [0] `tick_pending`; write `1` to clear (**W1C**). |
| | | `+0x04` | RO | **VALUE** — running counter sample. |

## Firmware build

**Vivado does not build the firmware.** You need a **RISC-V bare-metal toolchain** (e.g. `riscv32-unknown-elf-gcc` or `riscv64-unknown-elf-gcc`, or **clang with a working RISC-V triple and linker**). If you only have a prebuilt **`clock.mem`**, you can skip compilation and point simulation/synthesis at that file.

1. `cd firmware`
2. `make` or **`make all`** (default target builds the memory image)

**Outputs**

- **`../clock.mem`** (project root) — instruction ROM init for `imem` / Vivado `$readmemh`
- `firmware/build/clock.elf`, `clock.bin`, map file, optional disassembly listing from the Makefile

Digits are formatted with **packed ASCII tables and index shifts** (no division/modulo, no `printf`/`stdio`).

## Vivado simulation

1. Build firmware so **`clock.mem`** exists at the **project root** (same directory that contains `firmware/` and `rtl/`), **or** copy a known-good `clock.mem` there.
2. Create a simulation source set that includes:
   - `rtl/tb_basic.sv` (**simulation top**)
   - all **`rtl/*.sv`** files used by the DUT
   - **`rtl/picorv32.v`** (CPU core is Verilog, not SystemVerilog)
3. Set the **simulation run directory** in Vivado to that **project root** so `IMEM_HEX("clock.mem")` and `imem`’s `$readmemh` resolve.
4. Set top module to **`tb_basic`**.
5. Run **xsim**. The testbench self-checks UART text for **`MISSION CLOCK`**, **`00:00:00`**, and **`00:01:00`** (minute rollover) and finishes with **PASS** or **`$fatal`**.

## FPGA implementation and bring-up

1. **Synthesis / implementation top:** `rtl/pynq_z2_tx_demo_top.sv` (wraps `picorv32_soc_ref` with board clock and reset).
2. Add **`constraints/pynq_z2_tx_demo.xdc`** (or your course’s pin constraints if different).
3. Include **`clock.mem`** in the project so `imem` initializes correctly (same path semantics as simulation, relative to Vivado run / project settings).
4. Run synthesis → implementation → **Generate bitstream**.
5. **Program** the FPGA from Vivado Hardware Manager (or your lab flow).

SoC UART parameters in the FPGA top match **115200 baud** at the **post-PLL SoC clock** (see `pynq_z2_tx_demo_top.sv`).

## USB–UART wiring (demo XDC)

Use a **3.3 V** USB–UART adapter and a **common ground**:

| Signal | PYNQ-Z2 (this XDC) | Dongle |
|--------|---------------------|--------|
| GND | GND | **GND** |
| `uart_txd` | **W19** (PMODB) | Adapter **RXD** |
| `uart_rxd` | **W18** (PMODB) | Adapter **TXD** |

*TX of the board goes to RX of the dongle, and vice versa.*

## Firmware terminal UI (bonus)

The firmware uses **ANSI escape sequences** (colors and cursor positioning) for a small TUI: **clear screen once**, a green **“RISC-V MISSION CLOCK”** title, **cyan** time on a fixed row, **yellow** **STATUS: RUNNING**, and a **`|/-\` spinner** tied to the second count (bitwise index). Updates **rewrite fixed lines** instead of scrolling endless lines. If your terminal does not interpret ANSI, you may still see readable text with extra escape characters.
